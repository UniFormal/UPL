UPL vs MMT:

Pro UPL:

-when declaring structs, includes don't need to be stated explicitly:
In MMT:
total structure typesAsPredicates : ?SoftTypedTerms =
    include ?Terms❙
    include ?Propositions❙
    tp = term ⟶ prop❙
    of = [X,A] A X❙
  ❚
In UPL:
typesAsPredicates : SoftTypedTerms {
    type tp = term -> prop
    of = (X, A) -> (A X)
}


Pro MMT:

-curried and uncurried functions can be used interchangably
(in UPL uncurried version is needed for notation)

-type-checker is faster
(UPL struggles, especially for theories that include structs)
